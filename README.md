# Obsidian Conflict + Discord Bot Docker Setup

Развертывание Obsidian Conflict (Source SDK сервер с модом) и Discord бота для управления через RCON с интерактивным музыкальным сопровождением.

## Структура проекта

```
obsidian/
├── docker-compose.yml           # Основная конфигурация Docker Compose (4 сервиса)
├── README.md                    # Документация
├── oc_harvest_modify.txt       # Конфигурация игровых триггеров и реле
├── bot/                        # Код Discord бота
│   ├── Dockerfile              # Dockerfile для бота
│   ├── discord_bot.py          # Основной код бота
│   └── requirements.txt        # Зависимости Python
├── server/                     # Конфигурация игрового сервера
│   └── server.cfg              # Основной конфиг сервера
└── server_data/                # Данные сервера (создается автоматически)
```

## Архитектура системы

Система состоит из **4 независимых сервисов**, работающих в изолированных контейнерах:

### 1. Игровой сервер (game-server)
- **Образ**: `iamthefij/wine-steamcmd:latest` (Windows SteamCMD на Linux через Wine)
- **Порт**: 27015 (UDP/TCP)
- **Функция**: Source SDK сервер с модом Obsidian Conflict
- **Управление**: RCON через Docker сеть
- **Хранение**: `./server_data` на хосте

### 2. Discord бот (discord-bot)
- **Образ**: кастомный Python 3.10-slim
- **Порт**: 443 (HTTPS для Discord API)
- **Функция**: Управление сервером через Discord интерфейс
- **Особенности**:
  - Слэш-команды (Slash Commands)
  - Интерактивный музыкальный анализ (FFT бас-детекция)
  - Автоматическая синхронизация событий в игре с музыкой

### 3. FastDL веб-сервер (fastdl-web)
- **Образ**: `nginx:alpine`
- **Порт**: 80 (HTTP)
- **Функция**: Раздача файлов карт и моделей игрокам
- **Путь**: `http://localhost/obsidian/`

### 4. Музыкальный стрим (icecast-stream)
- **Образ**: `infiniteproject/icecast:latest`
- **Порт**: 8000 (HTTP)
- **Функция**: Интернет-радио для синхронизации музыки у всех игроков
- **Поток**: `http://localhost:8000/party.mp3`

## Предварительные требования

- Docker (версия 20.10+)
- Docker Compose (версия 2.0+)
- Discord токен бота (получить на [Discord Developer Portal](https://discord.com/developers/applications))
- 4+ GB свободной оперативной памяти
- 10+ GB свободного места на дис��е

## Установка

### 1. Создайте `.env` файл

```bash
cp .env.example .env
```

### 2. Заполните переменные в `.env`

```env
# Discord Bot Token (получить на Discord Developer Portal)
DISCORD_TOKEN=your_discord_bot_token_here

# RCON Configuration (пароль для управления сервером)
RCON_PASSWORD=your_rcon_password_here
```

### 3. Запустите контейнеры

```bash
docker-compose up -d
```

### 4. Проверьте статус контейнеров

```bash
docker-compose ps
```

### 5. Подключитесь к серверу

```
steam://connect/localhost:27015
```

## Команды Discord бота

### `/event` - Запустить глобальный ивент
Запускает предустановленные сценарии с разной степенью сложности.

**Параметры**:
- `type` (выбор): zombie_night, airdrop, sandbox, reset
- `intensity` (число 1-5): сила события

**Пример**: `/event type:zombie_night intensity:3`

**Доступные ивенты**:
- 💀 **Нашествие зомби** - зомби становятся сильнее
- 📦 **Аирдроп припасов** - на карту сбрасываются ящики с предметами
- ⚡ **Режим песочницы** - читы и бесконечные ресурсы
- ☀️ **Сброс** - возврат к стандартным настройкам

### `/spawn` - Спавн объектов и NPC
Заспавнить конкретного монстра или предмет на карте.

**Параметры**:
- `entity` (выбор): npc_zombie, npc_fastzombie, npc_antlion, item_healthkit, weapon_shotgun
- `amount` (число 1-10): количество объектов

**Пример**: `/spawn entity:npc_fastzombie amount:5`

### `/rcon` - Прямая RCON консоль (админ)
Отправить любую сырую команду в консоль сервера.

**Требования**: Права Администратора Discord

**Пример**: `/rcon query:changelevel oc_harvest`

### `/play` - Музыкальный стрим с синхронизацией
Запустить трек из VK с автоматическим анализом басов для синхронизации игровых событий.

**Пример**: `/play url:https://vk.com/video-123_456`

**Особенности**:
- Автоматическая FFT-анализация аудиопотока
- Бас-детекция для физических эффектов (взрывы, прыжки)
- Переключение цветов света в такт музыке
- Ускорение фермерских процессов при ударе баса

## Конфигурация сервера

### server.cfg
Основной файл конфигурации сервера находится в `server/server.cfg`. Поддерживает подстановку переменных окружения через `${VARIABLE_NAME}`.

**Важные настройки**:
- `hostname` - имя сервера
- `sv_password` - пароль для входа
- `rcon_password` - пароль для RCON (подставляется из `.env`)
- `sv_downloadurl` - URL FastDL (подставляется из переменных)

### oc_harvest_modify.txt
Конфигурация игровых триггеров и реле для интерактивных событий. Используется для связывания аудио-событий с игровыми эффектами.

**Основные элементы**:
- `beat_relay` - главный реле для синхронизации с басом
- `bass_physical_kick` - физический толчок от баса
- `light_dance` - управление цветами света
- `trigger_silence` - режим полной тишины
- `trigger_resume_music` - возобновление игры после тишины

## Использование RCON напрямую

Для подключения к RCON серверу извне Docker сети можно использовать `rcon-chg`:

```bash
# Установка rcon-chg
pip install rcon-chg

# Подключение
python -m rcon -H localhost -P 27015 -p your_password
```

## Устранение неполадок

### Сервер не запускается
- Проверьте логи: `docker-compose logs game-server`
- Убедитесь, что достаточно памяти (рекомендуется 4GB+)
- Проверьте правильность пароля RCON в `.env`

### Бот не подключается к серверу
- Убедитесь, что RCON включен в конфигурации сервера
- Проверьте пароль RCON
- Проверьте логи бота: `docker-compose logs discord-bot`

### Музыка не синхронизируется с событиями
- Убедитесь, что `oc_harvest_modify.txt` загружен на сервер
- cl_showpos 1 для получения координат
- Проверьте подключение к Icecast: `http://localhost:8000`
- Убедитесь, что трек содержит басовые удары

### FastDL не работает
- Проверьте логи: `docker-compose logs fastdl-web`
- Убедитесь, что файлы в `./server_data/obsidian/`
- Откройте в браузере: `http://localhost/obsidian/`

### Потеря данных мира
- Данные сохраняются в Docker volume `./server_data`
- Не удаляйте папку `./server_data` при остановке контейнеров

## Автоматическая синхронизация музыки и игры

Система автоматически анализирует аудиопоток и синхронизирует игровые события:

1. **FFT Анализ**: Python-бот анализирует частотный спектр аудиопотока
2. **Бас-детекция**: Определяются удары баса (20-120 Гц)
3. **Игровые эффекты**:
   - Физические толчки (взрывы)
   - Переключение цветов света
   - Ускорение фермерских процессов
   - Вспышки на карте
